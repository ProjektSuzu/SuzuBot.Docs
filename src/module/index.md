# 模块

本章节的内容将详细介绍各个模块中的功能和详细用法

## 命令前缀

命令前缀是指为了防止 SuzuBot 在日常聊天时被误触发而设置的关键词，使用命令时需要先输入该前缀

前缀与命令之间 **最好** 不要有任何空白

例如:
```
/test
```

上例中的 `/` 就是命令前缀。命令前缀还可以有以下几种方式
- 直接使用 SuzuBot 的名字 `铃酱`
- 在群聊中使用 At 的方式 `@铃酱`

因此，同一种命令可以有以下触发方式
```
/test
铃酱test
@铃酱test
```

为简化显示，在使用案例中仅会使用 `/` 作为命令前缀代表

部分命令可能不强制要求前缀，在使用案例中不会出现 `/`。但是使用时加不加都不影响命令的响应

例如
```
no_prefix
/no_prefix
```

## 命令参数

大部分命令都需要提供相关的参数。SuzuBot 的命令参数分为 可选参数 和 必选参数

各个参数的要求在命令说明处会有规定

### 可选参数

可选参数以方括号 `[]` 包围，使用命令时不填写该参数不影响命令的调用，但可能影响命令的结果

例如，对于以下命令格式
```
/test [参数]
```
下列两种格式均视为有效调用
```
/test
/test 123
```

可选参数包含多个的，除非有额外说明，使用位于位置靠后的参数时必须先填写位置靠前的参数

例如:
```
/test [参数1] [参数2]
```
跳过参数1而直接使用参数2是非法的

### 必选参数

必选参数以尖括号 `<>` 包围，使用命令时不填写该参数无法使用该命令

例如，对于以下命令格式
```
/test <参数>
```
只有以下一种格式视为有效调用
```
/test 123
```

::: tip
不论是可选参数还是必选参数，使用时候都不要把括号写进去
:::

## 命令权限

在部分命令或参数名称右上方，会有一个类似 <Badge type="tip" text="这样" vertical="middle" /> 的标签。它用于指示命令所需权限。

- 没有任何标签则说明任何人均可使用该命令
- <Badge type="warning" text="管理员" vertical="middle"/> 标签说明这个命令只有群组管理员才可以使用
- <Badge type="danger" text="开发者" vertical="middle"/> 标签说明这个命令只有开发者才可以使用

## 模块状态

在模块名称头上方，会有一个类似 <Badge type="tip" text="这样" vertical="middle" /> 的标签。它用于指示模块的状态。

- <Badge type="tip" text="稳定" vertical="middle"/> 说明该模块运行稳定，不会修改原先结构或者产生巨大变动
- <Badge type="danger" text="开发中" vertical="middle"/> 说明该模块处于频繁开发期，将会产生较多变动，且不保证最终成果与原版本相符合
- <Badge type="warning" text="默认关闭" vertical="middle"/> 说明该模块在群聊环境中默认处于关闭状态，使用该模块需要群组管理员手动开启
- <Badge type="warning" text="白名单" vertical="middle"/> 说明该模块需要白名单验证方可使用
